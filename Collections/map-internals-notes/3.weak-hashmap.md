## Internal Structure and Working

`WeakHashMap` stores key-value pairs in an array of buckets — structurally similar to `HashMap` — but wraps each **key** in a `WeakReference`. This allows the garbage collector to reclaim keys that have no other strong references, automatically removing the associated entry.

### How a put() works

1. The key is wrapped in a `WeakReference` linked to a **ReferenceQueue**.
2. The bucket index is computed from the key's `hashCode()`.
3. The entry is stored at that bucket index.

### How entry removal works

1. When no strong references to a key exist, the GC collects it.
2. The `WeakReference` wrapper is enqueued in the internal `ReferenceQueue`.
3. On the **next map operation** (`put`, `get`, `size`, etc.), `WeakHashMap` polls the queue and removes stale entries.

> Entry removal is **lazy** — it happens on the next map operation, not immediately when the GC runs.

### Node structure (simplified)

```java
private static class Entry<K,V> extends WeakReference<Object>
        implements Map.Entry<K,V> {

    V value;
    final int hash;
    Entry<K,V> next;

    Entry(Object key, V value, ReferenceQueue<Object> queue,
          int hash, Entry<K,V> next) {
        super(key, queue);  // key held as weak reference
        this.value = value;
        this.hash  = hash;
        this.next  = next;
    }
}
```

## Weak References Explained

Java has four reference strengths. Understanding them clarifies exactly when a `WeakHashMap` key is eligible for collection.

| Reference type | Class | GC behaviour |
|---------------|-------|--------------|
| Strong | Default (`Object o = new Object()`) | Never collected while reachable |
| Soft | `SoftReference<T>` | Collected only when JVM needs memory |
| Weak | `WeakReference<T>` | Collected at the **next GC cycle** once unreachable |
| Phantom | `PhantomReference<T>` | Collected after finalization |

`WeakHashMap` uses **weak references** for keys. Once the only remaining reference to a key object is the `WeakReference` inside the map, the key is eligible for collection at the next GC.

### Strong vs weak reference example

```java
Object key = new Object();         // strong reference
WeakReference<Object> weak = new WeakReference<>(key);

System.out.println(weak.get()); // the object

key = null;                        // drop the strong reference
System.gc();

System.out.println(weak.get()); // null — object was collected
```

## Entry Expiration and GC Behaviour

### Basic expiration

```java
WeakHashMap<Object, String> map = new WeakHashMap<>();

Object key = new Object();
map.put(key, "value");

System.out.println(map.size()); // 1
System.out.println(map.containsKey(key)); // true

key = null; // drop the only strong reference
System.gc();

// trigger a map operation so stale entries are purged
System.out.println(map.size()); // 0 (most likely)
```

> GC behaviour is not guaranteed to be immediate. In production, avoid depending on precise timing.

### String literal keys are never collected

```java
WeakHashMap<String, Integer> map = new WeakHashMap<>();
map.put("hello", 1); // "hello" is interned — strong reference held by JVM

System.gc();
System.out.println(map.size()); // 1 — "hello" is never collected
```

String literals are interned and held by the JVM's string pool, so they always have a strong reference. Use `new String("hello")` to create a key that can be collected.

```java
WeakHashMap<String, Integer> map = new WeakHashMap<>();
String key = new String("hello"); // not interned
map.put(key, 1);

key = null;
System.gc();
System.out.println(map.size()); // 0
```

## Null Support

- Allows **one null key** and **multiple null values**.
- The null key is handled specially — it is not wrapped in a `WeakReference` since `null` cannot be garbage collected.

```java
WeakHashMap<String, Integer> map = new WeakHashMap<>();
map.put(null, 42);
map.put("key", null);

System.out.println(map.get(null));  // 42
System.out.println(map.get("key")); // null
```

## WeakHashMap vs HashMap

| Feature | HashMap | WeakHashMap |
|---------|---------|-------------|
| Key reference type | Strong | Weak |
| Entry lifetime | Until explicitly removed | Until key is GC'd |
| Null keys | 1 allowed | 1 allowed |
| Ordering | None | None |
| Thread-safe | No | No |
| Predictable size | Yes | No — GC can shrink it at any time |
| Use case | General key-value lookup | Caches, metadata attached to objects |

## When to Use WeakHashMap

`WeakHashMap` is the right choice when the **map entry should not be the reason an object stays alive**.

### Good use cases

- **Object metadata caches** — attach computed data to objects without preventing their collection.
- **Listener or observer registries** — automatically clean up listeners when their owner is collected.
- **Canonicalization maps** — intern objects temporarily without causing memory leaks.

```java
// Attach rendering metadata to UI components
// without preventing components from being GC'd
WeakHashMap<Component, RenderState> renderCache = new WeakHashMap<>();

renderCache.put(button, new RenderState(...));

// When button is removed from UI and dereferenced,
// the renderCache entry is automatically removed too
```

### Bad use cases

- **Storing primitive wrapper keys** — boxed integers like `Integer.valueOf(n)` may be cached by the JVM and never collected.
- **String literal keys** — interned strings are never collected.
- **Thread-safe caches** — `WeakHashMap` is not thread-safe; use `Collections.synchronizedMap` or prefer `Caffeine` / `Guava`'s `CacheBuilder` for production caches.

## Pitfalls and Gotchas

### Pitfall 1: size() is unreliable

`size()` reflects the state at the time of the call. A GC between two `size()` calls can change the count.

```java
int s1 = map.size(); // 5
System.gc();
int s2 = map.size(); // 3 — entries were purged
```

### Pitfall 2: Iteration while GC runs

Iterating with a for-each and letting a GC happen mid-loop can cause entries to disappear. Always snapshot keys if you need a stable view.

```java
// Safer: snapshot the key set first
Set<MyKey> keys = new HashSet<>(map.keySet());
for (MyKey key : keys) {
    String value = map.get(key); // may still be null if GC ran
    if (value != null) {
        // process
    }
}
```

### Pitfall 3: Not thread-safe

`WeakHashMap` is not thread-safe. Wrap it when sharing across threads.

```java
Map<Object, String> syncMap =
        Collections.synchronizedMap(new WeakHashMap<>());
```

### Pitfall 4: Values holding strong references to keys

If the **value** holds a strong reference back to the key, the key can never be collected — the entry will never expire.

```java
// BAD: value references the key
map.put(key, new Wrapper(key)); // key stays alive via the value
```

## Time Complexity

| Operation | Average | Worst Case |
|-----------|---------|------------|
| `get(key)` | O(1) | O(n) |
| `put(key, value)` | O(1) | O(n) |
| `remove(key)` | O(1) | O(n) |
| `containsKey(key)` | O(1) | O(n) |
| `containsValue(value)` | O(n) | O(n) |
| Iteration | O(n) | O(n) |

> `WeakHashMap` does not treeify buckets (no Red-Black Tree conversion), so worst case remains O(n) unlike `HashMap`'s O(log n) in Java 8+.

## Common Coding Problems

### 1. Object metadata cache

Attach per-object metadata without causing memory leaks.

```java
WeakHashMap<Object, Map<String, Object>> metaStore = new WeakHashMap<>();

public void setMeta(Object obj, String key, Object value) {
    metaStore
        .computeIfAbsent(obj, k -> new HashMap<>())
        .put(key, value);
}

public Object getMeta(Object obj, String key) {
    Map<String, Object> meta = metaStore.get(obj);
    return meta != null ? meta.get(key) : null;
}
```

### 2. Auto-expiring listener registry

Listeners are removed automatically when their owner is garbage collected.

```java
WeakHashMap<Object, Runnable> listeners = new WeakHashMap<>();

public void register(Object owner, Runnable listener) {
    listeners.put(owner, listener);
}

public void fireAll() {
    for (Runnable listener : listeners.values()) {
        listener.run();
    }
}
```

### 3. Canonicalization map

Intern short-lived objects and let them expire naturally.

```java
WeakHashMap<String, WeakReference<MyObject>> pool = new WeakHashMap<>();

public MyObject intern(String key) {
    WeakReference<MyObject> ref = pool.get(key);
    MyObject obj = (ref != null) ? ref.get() : null;
    if (obj == null) {
        obj = new MyObject(key);
        pool.put(key, new WeakReference<>(obj));
    }
    return obj;
}
```

### 4. Counting live objects

Track how many instances of a class are currently reachable.

```java
WeakHashMap<MyClass, Boolean> liveInstances = new WeakHashMap<>();

public void track(MyClass instance) {
    liveInstances.put(instance, Boolean.TRUE);
}

public int liveCount() {
    return liveInstances.size(); // approximate — GC may have run
}
```
