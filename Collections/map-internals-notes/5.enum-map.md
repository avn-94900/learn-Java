## Internal Structure and Working

`EnumMap` is a specialised `Map` implementation where **keys must be enum constants from a single enum type**. It uses a plain **array** indexed by each constant's `ordinal()` value — no hashing, no buckets, no collision handling.

### How it works

- Internally holds two arrays of length equal to the number of enum constants: one for values, one for tracking which keys are present.
- The key's position in the array is simply `key.ordinal()`.
- No `hashCode()` or `equals()` is ever called on the key.

### Internal structure (simplified)

```java
// Inside EnumMap<K extends Enum<K>, V>
private final Class<K> keyType;
private transient K[] keyUniverse;   // all enum constants in ordinal order
private transient Object[] vals;     // vals[key.ordinal()] = value
private transient int size;
```

### How a put() works

1. Validate that the key belongs to the correct enum type.
2. Compute the slot: `index = key.ordinal()`.
3. Store the value at `vals[index]`.

### How a get() works

1. Compute `index = key.ordinal()`.
2. Return `vals[index]`.

No hashing, no probing, no tree traversal — just a direct array access.

```java
enum Day { MON, TUE, WED, THU, FRI, SAT, SUN }

EnumMap<Day, String> schedule = new EnumMap<>(Day.class);
schedule.put(Day.MON, "Standup at 9am");
schedule.put(Day.FRI, "Retrospective at 4pm");

System.out.println(schedule.get(Day.MON)); // Standup at 9am
```

## Basic Usage

### Constructors

```java
// Empty map for a given enum type
EnumMap<Day, String> map1 = new EnumMap<>(Day.class);

// Copy from another EnumMap of the same type
EnumMap<Day, String> map2 = new EnumMap<>(map1);

// Copy from a regular Map (must contain at least one entry to infer type)
Map<Day, String> source = new HashMap<>();
source.put(Day.MON, "Meeting");
EnumMap<Day, String> map3 = new EnumMap<>(source);
```

### Common operations

```java
enum Priority { LOW, MEDIUM, HIGH, CRITICAL }

EnumMap<Priority, List<String>> taskMap = new EnumMap<>(Priority.class);

// put and putIfAbsent
taskMap.put(Priority.HIGH, new ArrayList<>(List.of("Fix login bug")));
taskMap.putIfAbsent(Priority.LOW, new ArrayList<>());

// get with default
List<String> tasks = taskMap.getOrDefault(Priority.MEDIUM, Collections.emptyList());

// iteration — always in enum declaration order
for (Map.Entry<Priority, List<String>> entry : taskMap.entrySet()) {
    System.out.println(entry.getKey() + " → " + entry.getValue());
}
```

## Null Support

- **Null keys are not allowed** — throws `NullPointerException`.
- **Null values are allowed**.

```java
EnumMap<Day, String> map = new EnumMap<>(Day.class);

map.put(null, "value");  // NullPointerException
map.put(Day.MON, null);  // allowed — null value is valid

System.out.println(map.containsKey(Day.MON)); // true
System.out.println(map.get(Day.MON));         // null
```

> Use `containsKey()` to distinguish between a missing key and a key mapped to `null`.

## EnumMap vs HashMap

| Feature | HashMap | EnumMap |
|---------|---------|---------|
| Key type | Any object | Enum constants only |
| Internal structure | Bucket array + linked list / tree | Plain array indexed by ordinal |
| Hashing | `hashCode()` | None — direct array index |
| Ordering | None | Enum declaration order |
| Null keys | 1 allowed | Not allowed |
| Null values | Allowed | Allowed |
| Memory | Higher | Very compact |
| Performance | O(1) average | O(1) guaranteed, faster in practice |
| Thread-safe | No | No |
| Use case | General purpose | Enum-keyed mappings |

## EnumMap vs EnumSet

| Feature | EnumSet | EnumMap |
|---------|---------|---------|
| Purpose | Set of enum constants | Map from enum constants to values |
| Stores | Keys only | Key-value pairs |
| Internal structure | Bit vector | Array of values |
| Memory | Extremely compact | Compact |
| Use case | Membership checks, flag sets | Associating data with each enum constant |

```java
// EnumSet — which days are working days?
EnumSet<Day> workDays = EnumSet.range(Day.MON, Day.FRI);

// EnumMap — what is scheduled on each day?
EnumMap<Day, String> schedule = new EnumMap<>(Day.class);
schedule.put(Day.MON, "Standup");
```

## Ordering

`EnumMap` always iterates in the **natural order of enum constants** — the order they are declared in the enum definition. This is consistent and predictable regardless of insertion order.

```java
enum Season { SPRING, SUMMER, AUTUMN, WINTER }

EnumMap<Season, String> map = new EnumMap<>(Season.class);
map.put(Season.WINTER, "Cold");
map.put(Season.SPRING, "Warm");
map.put(Season.SUMMER, "Hot");

System.out.println(map.keySet());
// [SPRING, SUMMER, WINTER] — declaration order, not insertion order
```

## When to Use EnumMap

Use `EnumMap` whenever your keys are a **fixed, known set of enum constants**.

### Good use cases

- **State machines** — map each state to its handler or transitions.
- **Configuration per category** — map enum-based categories to settings.
- **Frequency or aggregation per enum** — count occurrences, sum values.
- **Dispatch tables** — map enum commands to `Runnable` or `Function`.

```java
// Dispatch table
enum Command { START, STOP, PAUSE, RESUME }

EnumMap<Command, Runnable> handlers = new EnumMap<>(Command.class);
handlers.put(Command.START,  () -> System.out.println("Starting..."));
handlers.put(Command.STOP,   () -> System.out.println("Stopping..."));
handlers.put(Command.PAUSE,  () -> System.out.println("Pausing..."));
handlers.put(Command.RESUME, () -> System.out.println("Resuming..."));

Command input = Command.START;
handlers.get(input).run(); // Starting...
```

### Bad use cases

- **Dynamic key sets** — if the set of keys is not known at compile time, use `HashMap`.
- **Mixed key types** — `EnumMap` only accepts a single enum type.

## Pitfalls and Gotchas

### Pitfall 1: Null key throws immediately

Unlike `HashMap`, putting a null key throws a `NullPointerException` at the `put()` call, not later.

```java
EnumMap<Day, String> map = new EnumMap<>(Day.class);
map.put(null, "oops"); // NullPointerException
```

### Pitfall 2: Size is always bounded by enum constants

`EnumMap` can hold at most as many entries as there are constants in the enum. You cannot put more entries than that.

```java
enum Coin { PENNY, NICKEL, DIME, QUARTER }
// max size of EnumMap<Coin, ?> is 4
```

### Pitfall 3: Not thread-safe

`EnumMap` is not thread-safe. Wrap with `Collections.synchronizedMap` when sharing across threads.

```java
Map<Day, String> syncMap =
        Collections.synchronizedMap(new EnumMap<>(Day.class));
```

### Pitfall 4: Constructing from an empty Map throws

When constructing from a non-`EnumMap` `Map`, the map must have at least one entry so that the enum type can be inferred.

```java
Map<Day, String> empty = new HashMap<>();
EnumMap<Day, String> map = new EnumMap<>(empty); // IllegalArgumentException
```

## Time Complexity

| Operation | Time |
|-----------|------|
| `get(key)` | O(1) — direct array access |
| `put(key, value)` | O(1) — direct array write |
| `remove(key)` | O(1) — direct array clear |
| `containsKey(key)` | O(1) |
| `containsValue(value)` | O(n) — scans the value array |
| Iteration | O(n) — n = number of enum constants |

> All O(1) operations are **true constant time** — there is no hashing overhead, no probing, and no tree traversal. `EnumMap` is the fastest `Map` implementation available for enum keys.

## Common Coding Problems

### 1. Frequency count by enum category

```java
enum Severity { LOW, MEDIUM, HIGH, CRITICAL }

public EnumMap<Severity, Integer> countBySeverity(List<Issue> issues) {
    EnumMap<Severity, Integer> counts = new EnumMap<>(Severity.class);
    for (Issue issue : issues) {
        counts.merge(issue.getSeverity(), 1, Integer::sum);
    }
    return counts;
}
```

### 2. State machine transitions

```java
enum State { IDLE, RUNNING, PAUSED, STOPPED }

EnumMap<State, EnumSet<State>> transitions = new EnumMap<>(State.class);
transitions.put(State.IDLE,    EnumSet.of(State.RUNNING));
transitions.put(State.RUNNING, EnumSet.of(State.PAUSED, State.STOPPED));
transitions.put(State.PAUSED,  EnumSet.of(State.RUNNING, State.STOPPED));
transitions.put(State.STOPPED, EnumSet.noneOf(State.class));

public boolean canTransition(State from, State to) {
    return transitions.getOrDefault(from, EnumSet.noneOf(State.class))
                      .contains(to);
}
```

### 3. Day-based scheduler

```java
enum Day { MON, TUE, WED, THU, FRI, SAT, SUN }

EnumMap<Day, List<String>> schedule = new EnumMap<>(Day.class);

public void addTask(Day day, String task) {
    schedule.computeIfAbsent(day, k -> new ArrayList<>()).add(task);
}

public List<String> getTasks(Day day) {
    return schedule.getOrDefault(day, Collections.emptyList());
}
```

### 4. Enum-based configuration

```java
enum Environment { DEV, STAGING, PROD }

EnumMap<Environment, String> dbUrls = new EnumMap<>(Environment.class);
dbUrls.put(Environment.DEV,     "jdbc:postgresql://localhost/dev");
dbUrls.put(Environment.STAGING, "jdbc:postgresql://staging-db/app");
dbUrls.put(Environment.PROD,    "jdbc:postgresql://prod-db/app");

public String getDbUrl(Environment env) {
    return dbUrls.get(env);
}
```

### 5. Aggregation with computeIfAbsent

```java
enum Department { ENGINEERING, MARKETING, SALES, HR }

public EnumMap<Department, List<String>> groupByDepartment(
        List<Employee> employees) {

    EnumMap<Department, List<String>> groups = new EnumMap<>(Department.class);
    for (Employee emp : employees) {
        groups.computeIfAbsent(emp.getDepartment(), k -> new ArrayList<>())
              .add(emp.getName());
    }
    return groups;
}
```
