## Internal Structure and Working

LinkedHashMap extends `HashMap` and adds a **doubly-linked list** running through all entries to maintain a predictable iteration order.

### How it differs from HashMap internally

- Every node is an `Entry<K,V>` that extends `HashMap.Node<K,V>` with two extra pointers: `before` and `after`.
- These pointers form a doubly-linked list across all entries, separate from the bucket chain.
- The list has a dedicated `head` (eldest entry) and `tail` (newest entry).

### Node structure

```java
static class Entry<K,V> extends HashMap.Node<K,V> {
    Entry<K,V> before;   // previous entry in iteration order
    Entry<K,V> after;    // next entry in iteration order
}
```

### How a put() works

1. Java calls `hashCode()` on the key — same as `HashMap`.
2. The bucket index is computed: `index = hash(key) & (n-1)`.
3. The entry is stored in the bucket array.
4. The new node is also **appended to the tail** of the doubly-linked list.

### Visual: bucket array + linked list

```
Bucket array          Doubly-linked list (iteration order)
[ 0 ] → Entry(A)      head → Entry(A) ↔ Entry(C) ↔ Entry(B) ← tail
[ 1 ] → null
[ 2 ] → Entry(C)
[ 3 ] → Entry(B)
```

Iteration always follows the doubly-linked list, not the bucket array.

## Insertion Order vs Access Order

LinkedHashMap supports two ordering modes controlled by the `accessOrder` constructor flag.

### Insertion Order (default)

- Entries are iterated in the order they were first inserted.
- Re-inserting an existing key does **not** move it — the original insertion position is kept.

```java
LinkedHashMap<String, Integer> map = new LinkedHashMap<>();
map.put("banana", 2);
map.put("apple", 1);
map.put("cherry", 3);

System.out.println(map.keySet());
// [banana, apple, cherry]

map.put("banana", 99); // update — position unchanged
System.out.println(map.keySet());
// [banana, apple, cherry]
```

### Access Order

- Enabled by passing `true` as the third constructor argument.
- Every `get()` or `put()` on an existing key moves that entry to the **tail** (most recently used).
- The `head` always holds the **least recently used** entry.
- This is the foundation for building an LRU cache.

```java
LinkedHashMap<String, Integer> map = new LinkedHashMap<>(16, 0.75f, true);
map.put("A", 1);
map.put("B", 2);
map.put("C", 3);

map.get("A"); // A moved to tail
System.out.println(map.keySet());
// [B, C, A]

map.get("B"); // B moved to tail
System.out.println(map.keySet());
// [C, A, B]
```

### Constructors

```java
// Insertion order (default)
LinkedHashMap<String, Integer> map1 = new LinkedHashMap<>();

// Insertion order, custom capacity and load factor
LinkedHashMap<String, Integer> map2 = new LinkedHashMap<>(32, 0.6f);

// Access order
LinkedHashMap<String, Integer> map3 = new LinkedHashMap<>(16, 0.75f, true);
```

### Ordering mode comparison

| Mode | Order maintained | Triggered by | Typical use |
|------|-----------------|--------------|-------------|
| Insertion order | First `put()` position | — | Ordered iteration |
| Access order | Most recently used at tail | `get()`, `put()` | LRU cache |

## Duplicate Keys

Behaviour is identical to `HashMap` — putting a duplicate key replaces the value and returns the previous one. The **iteration position of the key is not changed** in insertion-order mode.

```java
LinkedHashMap<String, Integer> map = new LinkedHashMap<>();
map.put("apple", 10);
map.put("banana", 20);
map.put("cherry", 30);

Integer previous = map.put("banana", 99);
System.out.println(previous); // 20
System.out.println(map);      // {apple=10, banana=99, cherry=30}
System.out.println(map.size()); // 3
```

### Related methods

| Method | Behavior |
|--------|----------|
| `put(k, v)` | Always inserts or replaces |
| `putIfAbsent(k, v)` | Inserts only if key is absent or mapped to null |
| `replace(k, v)` | Replaces only if key already exists |

## Null Support

- Allows **one null key** and **multiple null values** — same as `HashMap`.

```java
LinkedHashMap<String, Integer> map = new LinkedHashMap<>();
map.put(null, 1);
map.put("key", null);

System.out.println(map.get(null));  // 1
System.out.println(map.get("key")); // null
```

## LinkedHashMap vs HashMap

| Feature | HashMap | LinkedHashMap |
|---------|---------|---------------|
| Ordering | None | Insertion or access order |
| Internal structure | Bucket array + linked list per bucket | Bucket array + doubly-linked list across all entries |
| Memory | Less | More (two extra pointers per entry) |
| Iteration order | Unpredictable | Predictable |
| Null keys | 1 allowed | 1 allowed |
| Performance | Slightly faster | Marginally slower due to list maintenance |
| LRU cache support | No | Yes (with `accessOrder = true`) |
| Use case | General key-value lookup | Ordered iteration, LRU cache |

> Choose `LinkedHashMap` over `HashMap` only when iteration order matters or you need LRU behaviour. Otherwise the extra memory overhead is unnecessary.

## LRU Cache with removeEldestEntry

`LinkedHashMap` provides `removeEldestEntry(Map.Entry)` — a hook called after every `put()`. Override it to automatically evict the oldest entry when the map exceeds a size limit.

Combined with `accessOrder = true`, this produces a correct **Least Recently Used (LRU) cache** in a few lines.

```java
public class LRUCache<K, V> extends LinkedHashMap<K, V> {

    private final int capacity;

    public LRUCache(int capacity) {
        super(capacity, 0.75f, true); // accessOrder = true
        this.capacity = capacity;
    }

    @Override
    protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
        return size() > capacity; // evict head when over limit
    }
}
```

### Usage

```java
LRUCache<Integer, String> cache = new LRUCache<>(3);
cache.put(1, "A");
cache.put(2, "B");
cache.put(3, "C");

cache.get(1);       // 1 is now most recently used
cache.put(4, "D");  // evicts 2 (least recently used)

System.out.println(cache.keySet()); // [3, 1, 4]
```

### How eviction works step by step

| Step | Action | LRU order (head → tail) |
|------|--------|-------------------------|
| put(1) | Insert 1 | 1 |
| put(2) | Insert 2 | 1 → 2 |
| put(3) | Insert 3 | 1 → 2 → 3 |
| get(1) | Access 1, moved to tail | 2 → 3 → 1 |
| put(4) | Insert 4, evict head (2) | 3 → 1 → 4 |

## Time Complexity

| Operation | Average | Worst Case |
|-----------|---------|------------|
| `get(key)` | O(1) | O(log n) with trees, O(n) pre-Java 8 |
| `put(key, value)` | O(1) | O(log n) with trees |
| `remove(key)` | O(1) | O(log n) with trees |
| `containsKey(key)` | O(1) | O(log n) with trees |
| `containsValue(value)` | O(n) | O(n) |
| Iteration | O(n) | O(n) |
| Resize (rehash) | O(n) | O(n) |

> Iteration over `LinkedHashMap` is faster in practice than `HashMap` when the map is sparse, because it follows the doubly-linked list and skips empty buckets entirely.

## Common Coding Problems

### 1. First non-repeating character

LinkedHashMap preserves insertion order, making it straightforward to find the first character whose frequency is 1.

```java
public char firstNonRepeating(String s) {
    Map<Character, Integer> freq = new LinkedHashMap<>();
    for (char c : s.toCharArray()) {
        freq.put(c, freq.getOrDefault(c, 0) + 1);
    }
    for (Map.Entry<Character, Integer> entry : freq.entrySet()) {
        if (entry.getValue() == 1) return entry.getKey();
    }
    return '_';
}
```

### 2. LRU Cache (LeetCode 146)

```java
class LRUCache {

    private final int capacity;
    private final LinkedHashMap<Integer, Integer> cache;

    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.cache = new LinkedHashMap<>(capacity, 0.75f, true) {
            @Override
            protected boolean removeEldestEntry(Map.Entry<Integer, Integer> eldest) {
                return size() > capacity;
            }
        };
    }

    public int get(int key) {
        return cache.getOrDefault(key, -1);
    }

    public void put(int key, int value) {
        cache.put(key, value);
    }
}
```

### 3. Maintain insertion order while deduplicating

```java
public List<String> deduplicateOrdered(String[] words) {
    Map<String, Boolean> seen = new LinkedHashMap<>();
    for (String word : words) {
        seen.put(word, true);
    }
    return new ArrayList<>(seen.keySet());
}

// Input:  ["apple", "banana", "apple", "cherry", "banana"]
// Output: [apple, banana, cherry]
```

### 4. Group items preserving first-seen order

```java
public Map<Integer, List<String>> groupByLength(String[] words) {
    Map<Integer, List<String>> map = new LinkedHashMap<>();
    for (String word : words) {
        map.computeIfAbsent(word.length(), k -> new ArrayList<>()).add(word);
    }
    return map;
}

// Input:  ["hi", "hey", "ok", "bye", "no"]
// Output: {2=[hi, ok, no], 3=[hey, bye]}  — groups appear in order of first encounter
```

### 5. Recent activity log (fixed window)

Keep only the N most recently seen events using access-order eviction.

```java
public class RecentLog<K> extends LinkedHashMap<K, Long> {

    private final int maxSize;

    public RecentLog(int maxSize) {
        super(maxSize, 0.75f, true);
        this.maxSize = maxSize;
    }

    public void record(K event) {
        put(event, System.currentTimeMillis());
    }

    @Override
    protected boolean removeEldestEntry(Map.Entry<K, Long> eldest) {
        return size() > maxSize;
    }
}
```
