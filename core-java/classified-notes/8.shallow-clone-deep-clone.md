# Deep Copy in Java

## What is Deep Copy

A **deep copy** creates a completely independent clone of an object.

Changes to the copy do not affect the original.

A **shallow copy** only copies references. Both original and copy point to the same nested objects.

---

## Why It Matters

If `Employee` has an `Address` field, a shallow copy gives both objects the same `Address` reference.

Changing `emp2.address.city` would also change `emp1.address.city`.

Deep copy prevents this by creating a new `Address` for the copy.

---

## Option 1: Copy Constructor (Recommended)

The class itself handles copying. The client does not need to know internal structure.

**Address**

```java
class Address {
    String city;

    Address(String city) {
        this.city = city;
    }

    // Copy Constructor
    Address(Address other) {
        this.city = other.city;
    }
}
```

**Employee**

```java
class Employee {
    String name;
    Address address;

    Employee(String name, Address address) {
        this.name = name;
        this.address = address;
    }

    // Deep Copy Constructor
    Employee(Employee other) {
        this.name = other.name;
        this.address = new Address(other.address);
    }
}
```

**Usage**

```java
Employee emp1 = new Employee("Anil", new Address("Hyderabad"));
Employee emp2 = new Employee(emp1);

emp2.address.city = "Bangalore";

System.out.println(emp1.address.city); // Hyderabad
System.out.println(emp2.address.city); // Bangalore
```

---

## Option 2: clone() Method

Uses the `Cloneable` marker interface. Less preferred in modern Java.

**Address**

```java
class Address implements Cloneable {
    String city;

    Address(String city) {
        this.city = city;
    }

    @Override
    public Address clone() {
        return new Address(city);
    }
}
```

**Employee**

```java
class Employee implements Cloneable {
    String name;
    Address address;

    Employee(String name, Address address) {
        this.name = name;
        this.address = address;
    }

    @Override
    public Employee clone() {
        return new Employee(name, address.clone());
    }
}
```

**Usage**

```java
Employee emp2 = emp1.clone();
```

---

## How super.clone() Works

`Object.clone()` does a **shallow copy** by default.

1. `super.clone()` copies all fields, but nested objects are copied by reference.
2. You must manually clone each mutable nested object to make it deep.

```java
Employee cloned = (Employee) super.clone();
cloned.address = this.address.clone(); // Fix the shallow reference
```

---

## Comparison

| Approach | Recommended | Notes |
|---|---|---|
| Copy Constructor | Yes | Clean, safe, no checked exceptions |
| clone() with super.clone() | Sometimes | Verbose, needs Cloneable, throws checked exception |
| clone() with new object | Acceptable | Avoids super.clone() issues |
| Manual copy by client | No | Exposes internal structure |

---

## clone() Pitfalls

- Requires implementing the `Cloneable` **marker interface** (no methods, just a signal)
- `Object.clone()` throws `CloneNotSupportedException` if `Cloneable` is not implemented
- Default behavior is **shallow copy only**
- The return type is `Object`, requiring an explicit cast
- Easy to forget deep copying a nested mutable field

---

## Interview Answer

**Question: How do you implement deep copy in Java?**

1. Use a **copy constructor** (preferred).
2. In the copy constructor, create new instances of all mutable nested objects.
3. Never share references to mutable objects between original and copy.

```java
Employee(Employee other) {
    this.name = other.name;
    this.address = new Address(other.address);
}
```

This ensures changes to the copy do not affect the original.

---

## Key Terms

| Term | Meaning |
|---|---|
| Shallow Copy | Copies references, not the objects they point to |
| Deep Copy | Creates new instances of all nested mutable objects |
| Cloneable | Marker interface required for Object.clone() to work |
| Copy Constructor | Constructor that takes an object of the same type and copies its state |
| Mutable Object | An object whose internal state can change after creation |
