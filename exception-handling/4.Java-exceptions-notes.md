# Java Exceptions — Causes, Prevention & Handling

## RuntimeException (Unchecked)

Unchecked exceptions — compiler does not require them to be caught or declared. Usually indicate programming errors.

### Common RuntimeExceptions

| Exception | Cause | Prevention |
|-----------|-------|-----------|
| `NullPointerException` | Accessing method/field on null reference | Null checks, `Optional<T>`, `Objects.requireNonNull()` |
| `ArrayIndexOutOfBoundsException` | Index < 0 or >= array length | Check `arr.length` before access; use enhanced for-loop |
| `StringIndexOutOfBoundsException` | Invalid index on string | Validate index against `s.length()` |
| `ClassCastException` | Incompatible type cast | Use `instanceof` before casting; prefer generics |
| `IllegalArgumentException` | Invalid method argument | Validate parameters before use |
| `IllegalStateException` | Method called at wrong state | Ensure correct state transitions |
| `NumberFormatException` | Parsing malformed string as number | Validate with regex or try-catch before parsing |
| `ArithmeticException` | Illegal math (e.g. divide by zero) | Check divisor != 0 before dividing |
| `UnsupportedOperationException` | Operation not supported (e.g. mutating fixed-size list) | Check if collection is modifiable |
| `ConcurrentModificationException` | Modifying collection while iterating | Use `Iterator.remove()` or concurrent collections |
| `NegativeArraySizeException` | Creating array with negative size | Validate size is non-negative |
| `IndexOutOfBoundsException` | Invalid index on list | Check `list.size()` before access |
| `NoSuchElementException` | `next()` on empty iterator | Check `iterator.hasNext()` first |
| `EmptyStackException` | `pop()`/`peek()` on empty stack | Check `stack.isEmpty()` first |

**Examples:**
```java
String s = null;
s.length(); // NullPointerException

int[] arr = new int[3];
arr[5];     // ArrayIndexOutOfBoundsException

Object obj = "Java";
(Integer) obj; // ClassCastException

Integer.parseInt("abc"); // NumberFormatException

int a = 10 / 0; // ArithmeticException

List<String> list = Arrays.asList("a", "b");
list.add("c"); // UnsupportedOperationException

for (String s : list) {
    list.remove(s); // ConcurrentModificationException
}
```

## Multithreaded Environment Exceptions

### InterruptedException

Thrown when a thread is sleeping/waiting and another thread calls `interrupt()` on it.

```java
try {
    Thread.sleep(1000);
} catch (InterruptedException e) {
    Thread.currentThread().interrupt(); // always restore interrupt status
}
```

Never swallow this silently — it breaks cancellation.

---

### IllegalThreadStateException

Starting a thread that has already been started.

```java
Thread t = new Thread(() -> {});
t.start();
t.start(); // IllegalThreadStateException
```

Prevention: start a thread only once; check `Thread.State` if needed.

---

### ConcurrentModificationException

Modifying a shared collection while iterating without synchronization.

```java
// unsafe
for (String s : list) { list.remove(s); }

// safe options
Iterator<String> it = list.iterator();
while (it.hasNext()) { it.next(); it.remove(); }

// or use concurrent collection
List<String> safe = new CopyOnWriteArrayList<>();
```

---

### Deadlock (not an exception — freezes threads)

Two threads waiting on locks held by each other.

```java
synchronized (lock1) {
    synchronized (lock2) { ... } // Thread A
}
synchronized (lock2) {
    synchronized (lock1) { ... } // Thread B — deadlock
}
```

Prevention:
- Always acquire locks in a consistent global order.
- Use `ReentrantLock.tryLock(timeout, unit)` instead of `synchronized`.
- Avoid nested locking where possible.

Detect with: `jstack <pid>` or `jconsole`.

---

### Race Condition (not an exception — causes data corruption)

Two threads reading/writing shared state without synchronization.

```java
counter++; // not atomic — read-modify-write is 3 operations
```

Prevention:
```java
AtomicInteger counter = new AtomicInteger();
counter.incrementAndGet(); // atomic

// or
synchronized (this) { counter++; }

// or
volatile boolean flag = true; // for visibility only, not compound ops
```

---

### ThreadDeath (avoid)

Thrown internally when deprecated `Thread.stop()` is called. Subclass of `Error`.

Prevention: never use `stop()`, `suspend()`, `resume()` — all deprecated and unsafe. Use interrupt-based cooperative stopping instead.

## ExecutorService Exceptions

### RejectedExecutionException

Task submitted after executor is shut down, or task queue is full.

```java
executor.shutdown();
executor.submit(task); // RejectedExecutionException
```

Prevention:
```java
if (!executor.isShutdown()) executor.submit(task);

// or configure rejection policy in ThreadPoolExecutor
new ThreadPoolExecutor(..., new ThreadPoolExecutor.CallerRunsPolicy());
```

Rejection policies: `AbortPolicy` (default), `CallerRunsPolicy`, `DiscardPolicy`, `DiscardOldestPolicy`.

---

### ExecutionException

Wraps an exception thrown inside a submitted task. Retrieved via `Future.get()`.

```java
Future<Integer> f = executor.submit(() -> 1 / 0);
f.get(); // throws ExecutionException
```

Handling:
```java
try {
    f.get();
} catch (ExecutionException e) {
    Throwable cause = e.getCause(); // actual exception from the task
    if (cause instanceof NullPointerException) { ... }
    else if (cause instanceof SQLException) { ... }
}
```

---

### TimeoutException

`Future.get(timeout, unit)` did not complete in time.

```java
future.get(2, TimeUnit.SECONDS); // TimeoutException if task takes > 2s
```

Prevention: set realistic timeouts; monitor long-running tasks.

---

### CancellationException

Calling `get()` on a cancelled future.

```java
future.cancel(true);
future.get(); // CancellationException
```

Prevention: check `future.isCancelled()` before calling `get()`.

---

### Future.get() — Full Exception Handling Pattern

```java
Future<Integer> result = executor.submit(() -> loadDataFromDb());

try {
    Integer value = result.get(5, TimeUnit.SECONDS);
} catch (InterruptedException e) {
    Thread.currentThread().interrupt();
} catch (ExecutionException e) {
    Throwable cause = e.getCause();
    if (cause instanceof NullPointerException) {
        System.out.println("NPE: " + cause.getMessage());
    } else if (cause instanceof SQLException) {
        System.out.println("DB error: " + cause.getMessage());
    } else {
        System.out.println("Unexpected: " + cause);
    }
} catch (TimeoutException e) {
    System.out.println("Task timed out");
} catch (CancellationException e) {
    System.out.println("Task was cancelled");
}
```

## ExecutorService Exception Summary

| Exception | When | How to Handle |
|-----------|------|---------------|
| `RejectedExecutionException` | Submit after shutdown or queue full | Check `isShutdown()`; configure rejection policy |
| `InterruptedException` | Thread interrupted during `get()` / `awaitTermination()` | Restore interrupt status |
| `ExecutionException` | Task threw exception internally | Inspect `getCause()` |
| `TimeoutException` | `get(timeout)` exceeded | Increase timeout; monitor task |
| `CancellationException` | `get()` after `cancel()` | Check `isCancelled()` first |
| `NullPointerException` | Null task passed to `submit()` | Validate inputs |
| `IllegalArgumentException` | Invalid delay/time params | Validate parameters |

---

## General Prevention Strategies

| Strategy | Description |
|----------|-------------|
| Input validation | Validate all method inputs |
| Null checks / `Optional` | Avoid null references |
| Enhanced for-loop | Safer collection iteration |
| `try-catch` | Handle risky operations gracefully |
| Concurrent collections | `ConcurrentHashMap`, `CopyOnWriteArrayList` |
| `Atomic*` classes | Thread-safe counters and flags |
| `Thread.setUncaughtExceptionHandler` | Catch unhandled exceptions in threads |
| Static analysis tools | SonarLint, FindBugs catch risky code early |

**Graceful executor shutdown:**
```java
executor.shutdown();
if (!executor.awaitTermination(10, TimeUnit.SECONDS)) {
    executor.shutdownNow();
}
```
