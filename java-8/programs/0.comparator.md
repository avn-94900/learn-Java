# Comparable vs Comparator — Interview Notes

## Comparable

Used for:

> Default/Natural sorting

Interface:

```java id="79qg1c"
Comparable<T>
```

Method:

```java id="o56ezv"
compareTo()
```

Example:

```java id="ubx5b2"
class Employee implements Comparable<Employee> {

    int age;

    @Override
    public int compareTo(Employee e) {
        return Integer.compare(this.age, e.age);
    }
}
```

Usage:

```java id="k3f4yu"
Collections.sort(list);
```

---

### Key Points

* Sorting logic inside class
* Only one default sorting possible
* Used automatically by Java sorting methods
* Modifies class

---

## Comparator

Used for:

> Custom/Different sorting logics

Interface:

```java id="4vg4ir"
Comparator<T>
```

Method:

```java id="8sj6lq"
compare()
```

Example:

```java id="8wejmo"
Comparator<Employee> byAge =
    (e1, e2) -> Integer.compare(e1.age, e2.age);
```

Usage:

```java id="2w0pln"
Collections.sort(list, byAge);
```

or

```java id="k4hknf"
list.sort(byAge);
```

---

### Key Points

* Sorting logic outside class
* Multiple sorting logics possible
* No need to modify class
* More flexible

---

## Main Differences

| Comparable           | Comparator              |
| -------------------- | ----------------------- |
| Internal sorting     | External sorting        |
| compareTo()          | compare()               |
| One sorting logic    | Multiple sorting logics |
| Uses Comparable      | Uses Comparator         |
| Class changes needed | No class changes        |

---

## Modern Comparator Examples

### Sort by age

```java id="j0z1u5"
list.sort(
    Comparator.comparing(Employee::getAge)
);
```

---

### Reverse sorting

```java id="kl6a5g"
list.sort(
    Comparator.comparing(Employee::getAge)
              .reversed()
);
```

---

### Multiple fields

```java id="mjlwm2"
list.sort(
    Comparator.comparing(Employee::getDept)
              .thenComparing(Employee::getAge)
);

// ex-2
        l2.sort(
                Comparator.comparing(Employee::getSalary)
                          .reversed()
                          .thenComparing(Employee::getName)
        );

// ex-3
        l1.sort(
                Comparator.comparing(Employee::getName)
                          .thenComparing(
                              Comparator.comparing(Employee::getSalary)
                                        .reversed()
                          )
        );

// ex-4
        Comparator.comparing(Employee::getDept) // asc
          .thenComparing(Employee::getAge)   // asc
          .reversed()   // entire chain becomes reversed..
```

## Interview Questions

### Why Comparator when Comparable exists?

Because:

* multiple sorting rules needed
* runtime sorting needed
* cannot modify existing class

---

### Which is better?

Depends:

* Comparable → default sorting
* Comparator → flexible/custom sorting

Usually Comparator is used more in real projects.

---

## Important Interview Point

Avoid:

```java id="7y4vxh"
return a - b;
```

Use:

```java id="x1m24r"
Integer.compare(a, b);
```

because subtraction may cause overflow.

---

## Memory Trick

### Comparable

> Object compares itself

### Comparator

> Separate object compares objects

---

## One-Line Interview Answer

> Comparable provides default natural ordering using compareTo(), while Comparator provides custom sorting using compare(). Comparator is more flexible because we can create multiple sorting logics.
