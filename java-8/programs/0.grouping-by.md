# Java Streams — `groupingBy` + `partitioningBy` Layout

## 1. Basic `groupingBy`

```java
.collect(Collectors.groupingBy(
    classifierFunction
))
```

Example:

```java
Map<Character, List<String>> result =
    words.stream()
         .collect(Collectors.groupingBy(
             word -> word.charAt(0)
         ));
```

---

# 2. Group by Selected Field

## Group by department

```java

                 Employee::getDepartment

```

# 5. Custom Classifier Function

## Manual logic inside grouping

```java
Map<String, List<Employee>> result =
    employees.stream()
             .collect(Collectors.groupingBy(emp -> {
                 if(emp.getSalary() > 50000) {
                     return "HIGH";
                 }
                 return "LOW";
             }));
```

# 12. `partitioningBy`

## Always Boolean

```java
.collect(Collectors.partitioningBy(
    condition
))
```

---

# 13. Partition by Salary

```java

    employees.stream()
             .collect(Collectors.partitioningBy(
                 emp -> emp.getSalary() > 50000
             ));
```

# 16. Partition + Grouping

## First partition → then group

```java
Map<Boolean, Map<String, List<Employee>>> result =
    employees.stream()
             .collect(Collectors.partitioningBy(
                 emp -> emp.getSalary() > 50000,
                 Collectors.groupingBy(
                     Employee::getDepartment
                 )
             ));
```

# 17. Grouping Mistakes (Important)

## Mistake 1 — Using `map()` before grouping wrongly

Wrong:

```java
stream.map(Employee::getDepartment)
      .collect(Collectors.groupingBy(...))
```

You lose original object.

---

## Mistake 2 — Forgetting downstream collector

Wrong expectation:

```java
groupingBy(Employee::getDepartment)
```

Default result:

```java
Map<String, List<Employee>>
```

Not count/sum.

---

## Mistake 3 — Using `partitioningBy` for non-boolean

Wrong:

```java
partitioningBy(emp -> emp.getDepartment())
```

`partitioningBy` only supports:

```java
true / false
```

---

# 18. Most Important Patterns

## A. group → list

```java
groupingBy(key)
```

---

## B. group → count

```java
groupingBy(key, counting())
```

---

## C. group → sum

```java
groupingBy(key, summingInt(...))
```

---

## D. group → mapped values

```java
groupingBy(key,
    mapping(valueMapper, toList()))
```

---

## E. nested grouping

```java
groupingBy(key1,
    groupingBy(key2))
```

---

## F. partition → list

```java
partitioningBy(condition)
```

---

## G. partition → grouped

```java
partitioningBy(condition,
    groupingBy(key))
```

---

# Mental Model

```text
Stream
   ↓
Classifier Function
   ↓
groupingBy / partitioningBy
   ↓
Downstream Collector
   ↓
Final Map
```

---

# Final Structure to Remember

```java
groupingBy(
    classifier,
    downstreamCollector
)
```

```java
partitioningBy(
    predicate,
    downstreamCollector
)
```
