## Java 21 Features (LTS Release) — Interview Notes

Java 21 is a Long-Term Support (LTS) release focused on modern concurrency, scalability, developer productivity, and language improvements.

It introduces some of the most important JVM and concurrency upgrades since Java 8.

---

## 1. Virtual Threads (Finalized) — JEP 444

Virtual Threads are lightweight threads managed by the JVM instead of the operating system.

They allow applications to handle a very large number of concurrent tasks efficiently.

### Traditional Problem

Platform threads are expensive because:

* Each thread maps to an OS thread
* High memory usage
* Context switching overhead

### Virtual Thread Example

```java id="c7yx0v"
Thread.startVirtualThread(() -> {
    System.out.println("Running virtual thread");
});
```

### Executor Example

```java id="qwxrxs"
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> {
        System.out.println("Task executed");
    });
}
```

### Benefits

* Massive concurrency support
* Lower memory consumption
* Simpler async programming
* Easier than reactive programming for many use cases

### Important Interview Point

Virtual threads improve scalability for:

* REST APIs
* Database calls
* I/O-heavy applications

But they do not make CPU-heavy tasks faster.

## 2. Structured Concurrency (Preview) — JEP 453

Structured concurrency treats related concurrent tasks as a single unit.

### Problem Solved

Traditional thread management becomes difficult when:

* Multiple threads fail independently
* Cancellation handling is complex
* Error propagation is messy

### Example

```java id="h3l6u1"
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {

    Future<String> user = scope.fork(() -> fetchUser());
    Future<String> order = scope.fork(() -> fetchOrders());

    scope.join();
    scope.throwIfFailed();

    System.out.println(user.resultNow());
    System.out.println(order.resultNow());
}
```

### Benefits

* Better error handling
* Easier cancellation
* Cleaner concurrent code

---

## 3. Sequenced Collections — JEP 431

Introduces a common interface for collections with a defined order.

### New Methods

```java id="a2vtol"
list.getFirst();
list.getLast();

list.addFirst("A");
list.addLast("B");
```

### Supported Collections

* `List`
* `LinkedHashSet`
* `LinkedHashMap`

### Benefit

Provides consistent APIs for ordered collections.

---

## 4. Record Patterns — JEP 440

Allows direct extraction of values from records.

### Example

```java id="jn0mow"
record Person(String name, int age) {}

if (obj instanceof Person(String name, int age)) {
    System.out.println(name + " " + age);
}
```

### Benefit

* Cleaner data extraction
* Reduces boilerplate

---

## 5. Pattern Matching for Switch (Finalized) — JEP 441

Enhances switch statements with type patterns.

### Example

```java id="4m9e7m"
switch (obj) {
    case String s -> System.out.println("String: " + s);
    case Integer i -> System.out.println("Integer: " + i);
    case null -> System.out.println("Null value");
    default -> System.out.println("Unknown");
}
```

### Benefits

* Cleaner branching logic
* Safer type handling
* Better readability

---

## 6. String Templates (Preview) — JEP 430

Provides safer and more readable string interpolation.

### Example

```java id="dg5v5m"
String name = "Anil";

String message = STR."Hello \{name}";
```

### Benefits

* Easier string formatting
* Safer SQL/JSON template creation

### Important Note

This feature was preview-only in Java 21 and later removed from Java 23 for redesign.

---

## 7. Scoped Values (Preview) — JEP 446

Provides a safer alternative to `ThreadLocal`.

### Example

```java id="9d5f72"
static final ScopedValue<String> USER = ScopedValue.newInstance();

ScopedValue.where(USER, "Anil")
        .run(() -> {
            System.out.println(USER.get());
        });
```

### Benefits

* Better for virtual threads
* Immutable context sharing
* Avoids ThreadLocal memory issues

---

## 8. Unnamed Patterns and Variables (Preview) — JEP 443

Allows ignoring unused variables using `_`.

### Example

```java id="e4q7wx"
if (obj instanceof User(_, int age)) {
    System.out.println(age);
}
```

### Benefit

Makes code cleaner when some values are unnecessary.

---

## 9. Foreign Function & Memory API (Preview) — JEP 442

Modern replacement for JNI.

### Capabilities

* Call native libraries directly
* Access off-heap memory safely

### Benefits

* Better performance
* Safer than JNI
* Easier native integration

---

## 10. ZGC and Generational ZGC Improvements

Java 21 improves low-latency garbage collection.

### Benefits

* Better pause times
* Improved scalability
* Better memory management for large applications

---

## Important Interview Insight

A common assumption is:

> "Java 21 is mainly syntax improvements."

That is incomplete.

The biggest impact of Java 21 is actually:

* Virtual Threads
* Structured Concurrency
* JVM scalability improvements

These features significantly change how high-concurrency Java applications can be designed.

## Java 21 Focus Areas

Java 21 mainly improves:

| Area                   | Improvements                            |
| ---------------------- | --------------------------------------- |
| Concurrency            | Virtual Threads, Structured Concurrency |
| Language               | Record Patterns, Pattern Matching       |
| Collections            | Sequenced Collections                   |
| Native Integration     | Foreign Function API                    |
| Performance            | ZGC Improvements                        |
| Developer Productivity | String Templates, Scoped Values         |

---

## Most Important Features for Interviews

If interviewers ask about Java 21, focus mainly on:

1. Virtual Threads
2. Structured Concurrency
3. Pattern Matching for Switch
4. Record Patterns
5. Scoped Values
6. Sequenced Collections

These are the most discussed enterprise-level changes.

---

## Quick Summary

Java 21 is important because it modernizes Java concurrency.

Main highlights:

* Lightweight threads using Virtual Threads
* Cleaner concurrent programming
* Better pattern matching
* Improved JVM scalability
* Modern native memory access

It is one of the most significant LTS releases after Java 8.
