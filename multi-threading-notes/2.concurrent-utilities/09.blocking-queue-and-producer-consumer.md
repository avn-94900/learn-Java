# BlockingQueue & Producer-Consumer — Java

## What is BlockingQueue?

`BlockingQueue` is a thread-safe interface in `java.util.concurrent` designed for producer-consumer scenarios.

**Key traits:**
- Thread-safe — multiple threads can access concurrently
- Blocking operations — auto-waits when queue is full or empty
- Can be bounded (fixed size) or unbounded

**Interface hierarchy:**
```
java.util.Collection<E>
    ↳ java.util.Queue<E>
        ↳ java.util.concurrent.BlockingQueue<E>
```

---

## Core Methods

| Category | Method | Behavior |
|----------|--------|----------|
| Blocking | `put(E e)` | Waits if full |
| Blocking | `take()` | Waits if empty |
| Non-blocking | `offer(E e)` | Returns `false` if full |
| Non-blocking | `poll()` | Returns `null` if empty |
| Timed | `offer(E e, long t, TimeUnit u)` | Waits up to timeout |
| Timed | `poll(long t, TimeUnit u)` | Waits up to timeout |
| Utility | `remainingCapacity()` | Space left in queue |
| Utility | `drainTo(Collection c)` | Bulk remove to collection |

## Implementations

### ArrayBlockingQueue
Bounded, array-backed, FIFO.
```java
BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(10);

// With fairness (fair thread ordering)
new ArrayBlockingQueue<>(100, true);
```
Use when: capacity is known upfront.

### LinkedBlockingQueue
Optionally bounded, linked-node, FIFO. Higher throughput.
```java
BlockingQueue<String> unbounded = new LinkedBlockingQueue<>();
BlockingQueue<String> bounded   = new LinkedBlockingQueue<>(100);
```
Use when: high throughput or dynamic sizing needed.

### PriorityBlockingQueue
Unbounded, priority-ordered (via `Comparable` or `Comparator`).
```java
BlockingQueue<Task> queue = new PriorityBlockingQueue<>(10,
    (t1, t2) -> Integer.compare(t1.getPriority(), t2.getPriority()));
```
Use when: elements need priority-based processing.

### DelayQueue
Unbounded. Elements available only after their delay expires. Elements must implement `Delayed`.
```java
BlockingQueue<DelayedTask> queue = new DelayQueue<>();
```

```java
class DelayedTask implements Delayed {
    private final long expire;
    private final String task;

    public DelayedTask(String task, long delayMs) {
        this.task = task;
        this.expire = System.currentTimeMillis() + delayMs;
    }

    @Override
    public long getDelay(TimeUnit unit) {
        return unit.convert(expire - System.currentTimeMillis(), TimeUnit.MILLISECONDS);
    }

    @Override
    public int compareTo(Delayed o) {
        return Long.compare(this.expire, ((DelayedTask) o).expire);
    }
}
```
Use when: scheduled/delayed task processing.

### SynchronousQueue
Zero capacity — direct hand-off between threads.
```java
BlockingQueue<String> queue = new SynchronousQueue<>();
```
Use when: direct thread-to-thread transfer, no buffering needed.

### LinkedTransferQueue
Unbounded with extra `transfer()` methods for direct handoffs.
```java
TransferQueue<String> queue = new LinkedTransferQueue<>();
// transfer(e)        — blocks until consumer takes it
// tryTransfer(e)     — immediate, returns false if no consumer
```
Use when: you need both queuing and direct-transfer semantics.

## Performance Summary

| Implementation | Throughput | Memory | Notes |
|----------------|-----------|--------|-------|
| `LinkedBlockingQueue` | Highest | Dynamic | Best general-purpose |
| `ArrayBlockingQueue` | Moderate | Fixed | Predictable memory |
| `PriorityBlockingQueue` | Lower | Dynamic | Sorting overhead |
| `SynchronousQueue` | Highest (direct) | Minimal | No buffering |

## Producer-Consumer Patterns

### Approach 1 — `wait()` / `notify()` (Classic)

Threads coordinate via a shared lock. One waits while the other works.

```java
class SharedQueue {
    private int data;
    private boolean available = false;

    public synchronized void produce(int value) throws InterruptedException {
        while (available) {
            wait(); // wait until consumer consumes
        }
        data = value;
        System.out.println("Produced: " + data);
        available = true;
        notify(); // notify consumer
    }

    public synchronized int consume() throws InterruptedException {
        while (!available) {
            wait(); // wait until producer produces
        }
        System.out.println("Consumed: " + data);
        available = false;
        notify(); // notify producer
        return data;
    }
}

class Producer extends Thread {
    private SharedQueue queue;
    Producer(SharedQueue queue) { this.queue = queue; }

    public void run() {
        for (int i = 1; i <= 5; i++) {
            try {
                queue.produce(i);
                Thread.sleep(500);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }
}

class Consumer extends Thread {
    private SharedQueue queue;
    Consumer(SharedQueue queue) { this.queue = queue; }

    public void run() {
        for (int i = 1; i <= 5; i++) {
            try {
                queue.consume();
                Thread.sleep(500);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }
}

public class ProducerConsumerDemo {
    public static void main(String[] args) {
        SharedQueue queue = new SharedQueue();
        new Producer(queue).start();
        new Consumer(queue).start();
    }
}
```

**Output:**
```
Produced: 1
Consumed: 1
Produced: 2
Consumed: 2
...
```

### Approach 1b — Same `SharedQueue`, Lambda Style (Java 8+)

No change to `SharedQueue`. Replace `Producer`/`Consumer` classes with lambdas.

```java
public class LambdaProducerConsumer {
    public static void main(String[] args) {
        SharedQueue queue = new SharedQueue();

        Runnable producer = () -> {
            for (int i = 1; i <= 5; i++) {
                try {
                    queue.produce(i);
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        };

        Runnable consumer = () -> {
            for (int i = 1; i <= 5; i++) {
                try {
                    queue.consume();
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        };

        new Thread(producer, "Producer-Thread").start();
        new Thread(consumer, "Consumer-Thread").start();
    }
}
```

### Approach 2 — `BlockingQueue` + Lambda (Modern, Recommended)

No manual `wait/notify` or `synchronized` needed. `BlockingQueue` handles all coordination.

```java
import java.util.concurrent.ArrayBlockingQueue;
import java.util.concurrent.BlockingQueue;

public class Main {
    public static void main(String[] args) {
        BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(1);

        Runnable producer = () -> {
            for (int i = 1; i <= 5; i++) {
                try {
                    queue.put(i);             // blocks if full
                    System.out.println("Produced: " + i);
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        };

        Runnable consumer = () -> {
            for (int i = 1; i <= 5; i++) {
                try {
                    int v = queue.take();     // blocks if empty
                    System.out.println("Consumed: " + v);
                    Thread.sleep(2000);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        };

        new Thread(producer, "Producer").start();
        new Thread(consumer, "Consumer").start();
    }
}
```

**Output:**
```
Produced: 1
Consumed: 1
Produced: 2
Consumed: 2
...
```

### Approach 3 — Multiple Producers & Consumers

```java
import java.util.concurrent.*;

public class MultiProducerConsumer {
    private static final BlockingQueue<String> queue = new LinkedBlockingQueue<>(50);

    static class Producer implements Runnable {
        private final String name;
        Producer(String name) { this.name = name; }

        @Override
        public void run() {
            try {
                for (int i = 1; i <= 10; i++) {
                    String item = name + "-Item-" + i;
                    queue.put(item);
                    System.out.println(name + " produced: " + item);
                    Thread.sleep(200);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }

    static class Consumer implements Runnable {
        private final String name;
        Consumer(String name) { this.name = name; }

        @Override
        public void run() {
            try {
                while (true) {
                    String item = queue.take();
                    System.out.println(name + " consumed: " + item);
                    Thread.sleep(300);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }

    public static void main(String[] args) {
        for (int i = 1; i <= 3; i++) new Thread(new Producer("Producer-" + i)).start();
        for (int i = 1; i <= 2; i++) new Thread(new Consumer("Consumer-" + i)).start();
    }
}
```

### Anti-pattern — `Thread.sleep()` Only (No Coordination)

```java
class WrongQueue {
    private int data;
    private boolean produced = false;

    public void produce(int value) {
        if (!produced) {
            data = value;
            System.out.println("Produced (Wrong): " + value);
            produced = true;
        }
    }

    public void consume() {
        if (produced) {
            System.out.println("Consumed (Wrong): " + data);
            produced = false;
        }
    }
}
```

**Problem:** No synchronization → data can be missed or consumed out of order.

```
Produced (Wrong): 1
Produced (Wrong): 2
Consumed (Wrong): 2   ← item 1 was lost
```

## Approach Comparison

| Feature | `wait()/notify()` | `BlockingQueue` |
|---------|-------------------|-----------------|
| Communication | Manual, coordinated | Built-in |
| Synchronization | Requires `synchronized` | Handled internally |
| Thread-safe | Yes | Yes |
| Boilerplate | High | Low |
| Recommended for | Learning / legacy | Production code |

## Best Practices

**Handle `InterruptedException` correctly:**
```java
try {
    queue.put(item);
} catch (InterruptedException e) {
    Thread.currentThread().interrupt(); // restore interrupt status
}
```

**Graceful shutdown with timed poll:**
```java
private static volatile boolean shutdown = false;

// In consumer loop:
while (!shutdown) {
    String item = queue.poll(1, TimeUnit.SECONDS); // don't block forever
    if (item != null) process(item);
}
// Drain remaining items after shutdown signal
while (!queue.isEmpty()) {
    process(queue.poll());
}
```

**Monitor queue health:**
```java
queue.size();               // current elements
queue.remainingCapacity();  // space left
queue.isEmpty();            // is empty?
```

## Common Pitfalls

- **Unbounded queue + fast producer** → `OutOfMemoryError`. Prefer bounded queues in production.
- **Null elements** → all `BlockingQueue` implementations throw `NullPointerException` on `null`.
- **Deadlock** → avoid circular put dependencies across two queues between two threads.
